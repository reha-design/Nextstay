# Nextstay 예약 시스템 및 핵심 비즈니스 로직 구현 결과

숙소 및 객실 관리를 넘어, 실제 예약이 이루어지는 **핵심 예약 엔진**과 **동시성 제어 시스템**을 성공적으로 구현하고 검증을 완료했습니다.

## 🚀 추가된 주요 기능

### 1. 예약(Booking) 시스템 프로세스
- **원스톱 예약**: 게스트가 날짜를 선택하면 서버가 실시간으로 할인/성수기 가격을 재계산하고 예약을 생성합니다.
- **예약 상태 관리**: `CONFIRMED`, `PENDING`, `CANCELLED` 등 상태 기반의 예약 라이프사이클을 관리합니다.
- **예약 및 캘린더 중복 방지**: 이미 예약된 날짜에 대해서는 중복 예약을 원천 차단합니다.

### 2. 견고한 동시성 제어 (Concurrency Control)
- **비관적 락(`PESSIMISTIC_WRITE`)**: 동일 객실에 대해 여러 명의 사용자가 동시에 예약 버튼을 누르더라도, 데이터베이스 레벨에서 락을 획득하여 단 한 명만 예약에 성공하도록 보장합니다.
- **트랜잭션 격리 최적화**: MySQL의 `REPEATABLE_READ` 격리 수준에서 발생할 수 있는 데이터 가시성 문제를 해결하기 위해, 예약 로직에 `READ_COMMITTED` 격리 수준을 적용했습니다.

### 3. 실시간 가격 계산 엔진 (Refined)
- **정밀한 금융 연산**: `BigDecimal`을 도입하여 부동소수점 오차 없는 정확한 결제 금액을 산출합니다.
- **성수기 + 연박 할인**: 성수기 가중치와 장기 숙박 할인을 복합적으로 적용하는 엔진을 구현했습니다.

## ✅ 검증 및 테스트 결과

### 1. 통합 검증 (`test_api.py`)
- 회원가입부터 숙소 등록, 가격 계산, 예약 생성까지의 전체 시나리오가 정상 작동함을 확인했습니다.

### 2. 동시성 부하 테스트 (`test_load.py`)
- 5명의 게스트가 0.1초 내외의 간격으로 **동일한 객실/날짜에 예약**을 시도하는 공격적인 시나리오를 수행했습니다.
- **결과**: 단 1명만 예약 성공(201), 나머지 4명은 예약 충돌(409) 메시지를 받으며 데이터 무결성이 유지됨을 증명했습니다.
