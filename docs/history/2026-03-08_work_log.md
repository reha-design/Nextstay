# ✅ 2026-03-08 RabbitMQ & 분석 서비스 구현 완료 리포트

요청하신 아키텍처 설계에 따라 RabbitMQ 비동기 처리 고도화 및 분석 서비스 기능 구현을 완료하였습니다.

## 🛠️ 작업 내용

### 1. 백엔드 (Kotlin/Spring Boot) - RabbitMQ 고도화
- **RabbitMqConfig.kt**: 
  - 메시지 처리 실패 시 유실을 방지하기 위한  **Dead Letter Queue(DLQ)**  및  **Dead Letter Exchange(DLX)**  설정을 추가하였습니다.

  
  #### 🛠️ DLQ/DLX 상세 동작 원리
  비동기 메시징 시스템에서 특정 메시지가 처리되지 못하고 계속 실패할 경우, 시스템 전체의 병목이 되거나 데이터가 유실될 수 있습니다. 이를 방지하기 위해 다음과 같이 구성했습니다:

  ```mermaid
  graph LR
      MainEx["Main Exchange<br>(Direct)"] -->|reservation.created| MainQ["Main Queue<br>(reservation.queue)"]
      
      MainQ -- "1. 실패 (nack/reject)" --> DLX["Dead Letter Exchange<br>(Direct)"]
      DLX -->|reservation.dead| DLQ["Dead Letter Queue<br>(reservation.dlq)"]
      
      subgraph "Error Handling Flow"
      DLX
      DLQ
      end
  ```

  1.  **동작 조건**  : `reservation.queue`에서 메시지 처리가 실패하고(Exception 발생), 더 이상 재시도(Retry)하지 않기로 결정되면 RabbitMQ는 해당 메시지를 설정된 `x-dead-letter-exchange`(`reservation.dlx`)로 보냅니다.
  2.  **라우팅**  : `reservation.dlx`는 받은 메시지를 `x-dead-letter-routing-key`(`reservation.dead`)에 따라 `reservation.dlq`로 전달합니다.
  3.  **사후 조치**  : `reservation.dlq`에 쌓인 메시지는 관리자가 나중에 원인을 분석하여 수동으로 다시 처리하거나, 별도의 에러 처리 워커가 로그를 남기고 종료할 수 있습니다.
  4.  **장점**  : 메인 큐의 흐름을 방해하지 않으면서도  **실패한 데이터를 안전하게 격리**  하여 시스템 가용성을 높입니다.

- **ReservationEventConsumer.kt**: 
  - 단순 로그 출력을 넘어, 알림톡 발송 및 예약 상태 업데이트를 시뮬레이션하는 비즈니스 로직을 구현하였습니다.
  - 예외 발생 시 스택 트레이스를 기록하고 메시지를 DLQ로 넘길 수 있도록 에러 핸들링을 강화했습니다.

### 2. 분석 서비스 (Bun/Elysia.js) - SQLite 영속성 강화
- **index.ts**:
  -  **SQLite WAL 모드 도입**  : `analytics.sqlite` 파일을 생성하고 WAL(Write-Ahead Logging) 모드를 활성화하여 쓰기 지연 없는 고성능 로그 수집 환경을 구축했습니다.
  -  **방문자 로깅 (`/stats/v1/visits`)**  : 이제 모든 방문 로그가 `visit_logs` 테이블에 영구 저장됩니다.
  -  **이벤트 로깅 (`/analytics/event`)**  : 커스텀 이벤트 역시 `event_logs` 테이블에 자동 축적됩니다.
  -  **실시간 통계 (`/stats/v1/visits/today`)**  : 더미 데이터 대신  **실제 DB 데이터**  를 쿼리하여 오늘 방문자 수와 인기 경로 TOP 5를 반환합니다.


## 🔍 검증 방법 (Verification)

### [RabbitMQ] 비동기 처리 및 DLQ 확인
1. **이벤트 발행**: 메인 서버에서 예약 이벤트를 발행합니다.
2. **로그 확인**: `ReservationEventConsumer` 로그에 `>>> [비동기 예약 이벤트 수신]` 문구와 함께 알림톡 발송 과정이 출력되는지 확인합니다.
3. **DLQ 동작**: (테스트 시) 인위적으로 예외를 발생시키면 메시지가 `reservation.dlq`로 이동하는지 RabbitMQ 매니지먼트 콘솔에서 확인할 수 있습니다.
