# Architecture Decision Record: 대용량 트래픽 대응을 위한 '지연 발급(Lazy Issue)' 쿠폰 시스템 도입

## 1. 배경 (Context)

미스터멘션(Nextstay) 플랫폼의 성장에 따라, 전체 회원(수십만~수백만 명)을 대상으로 하는 대규모 쿠폰 발급이나 프로모션 이벤트가 잦아질 것으로 예상됩니다.
전통적인 방식인 **'Push 모델'**(이벤트 대상 유저 수만큼 DB의 유저-쿠폰 매핑 테이블에 레코드를 삽입하는 방식)은 이벤트 시작 시점에 메인 데이터베이스(MySQL)에 엄청난 트랜잭션 부하(I/O)를 발생시킵니다. 이는 핵심 비즈니스 로직인 타 사용자의 예약 및 결제 트랜잭션까지 마비시킬 수 있는 치명적인 위험 요인입니다. 
또한 전체 쿠폰 소지자 중 실제 결제까지 이어지는 사용자의 전환율(통상 10% 미만)을 고려할 때, 90% 이상의 미사용 스레기 데이터가 DB 스토리지 용량을 낭비하는 문제도 존재합니다.

## 2. 결정 (Decision)

전체 회원을 대상으로 쿠폰을 지급할 때 개별 유저별 데이터를 즉시 생성(Push)하지 않고, **쿠폰 마스터(Master) 정보만 등록해 둔 뒤 사용자가 실제로 결제 화면에서 쿠폰을 사용할 때만 유효성 검증과 사용 이력을 기록하는 '지연 발급(Lazy Issue) / 규칙 기반(Rule-based)' 아키텍처**를 도입합니다.

## 3. 도입 근거 (Rationale)

### 3.1. 압도적인 데이터베이스 부하 최적화 (O(N) ➡ O(1))
- **이벤트 시작 시**: 수백만 건의 `INSERT` 대신, 관리자가 생성한 정책(타겟, 할인율, 만료일 등)이 포함된 마스터 쿠폰 데이터 **단 1건**만 저장소에 `INSERT` 됩니다. DB 마비 위험이 사라집니다.
- **쿠폰 노출 시**: 사용자가 쿠폰함을 조회할 때, 개별 소유 테이블을 조회하는 대신 마스터 테이블에서 현재 활성화된(규칙에 부합하는) 전체 쿠폰을 노출합니다.

### 3.2. 저장 공간 절약 및 가성비
- 오직 결제를 진행하며 **쿠폰을 소모(Consume)한 사용자(전환된 10%의 사용자)에 대해서만** `user_coupon_usage` (사용 이력 테이블)에 단 1줄의 기록이 남습니다. 무의미한 90%의 오크우폰 데이터가 DB를 차지하는 것을 원천적으로 차단합니다.

### 3.3. 알림 서버 (Worker) 부하 최소화 및 타겟팅
- **리마인드 알림 마케팅 최적화**: "만료 3일 전, 아직 쿠폰을 쓰지 않은 유저에게 알림 발송" 상황을 가정했을 때, 기존 시스템이라면 90만 건의 쿠폰 테이블을 뒤져야 합니다.
- **지연 발급의 해결책 (Anti-Join)**: 전체 대상 유저 풀에서 `user_coupon_usage`에 기록된 10%의 유저 아이디만 인덱스로 걸러낸(MINUS) 후 나머지 90%에게 알림을 발송하게 됩니다. 쓸모없는 90만 줄의 개별 쿠폰 정보를 관리하는 부담(수정, 상태 관리 등)을 없애고 쿼리를 가볍게 유지할 수 있습니다.
- 백그라운드 Worker와 메시지 큐(RabbitMQ)를 연동하여 알림 트래픽 병목 역시 비동기로 안전하게 처리합니다.

## 4. 결과 및 트레이드오프 (Consequences)

### 장점 (Positive)
- **높은 안정성**: 대규모 트래픽 이벤트에서도 예약(핵심 도메인)에 지장을 주지 않는 완벽한 격리와 성능.
- **마케팅 유연성 향상**: 마스터 구조를 차용하므로 "선착순 1만 명"과 같이 Redis 등을 결합한 실시간 마케팅 통제가 압도적으로 편해집니다.

### 약점 및 보완책 (Negative / Risks)
- **개인화 마케팅 패러다임 변화**: 사용자의 지갑(Wallet)에 물리적인 쿠폰 row가 들어있지 않으므로, 유저 단위를 기반으로 한 상태 관리(예: "당신만을 위한 쿠폰 취소")가 상대적으로 직관적이지 않을 수 있습니다.
- **해결 방안**: 쿠폰 마스터에 대상 타겟 군(Segment IDs) 조건을 명확히 설계하여, 쿼리 시 유저의 프로필 속성과 마스터 쿠폰의 룰(Rule)을 조인(Join)하여 평가하는 'Rule Engine' 형태로 고도화할 계획입니다.
