# Nextstay 메인 페이지 아키텍처 및 기획안

이 문서는 기존 미스터멘션 메인 페이지(`ex.html`)의 단점(단방향 접근, SP/jQuery 혼용 스파게티 코드, 비반응형 구조)을 개선하기 위한 **Nuxt 3 기반 Nextstay 프로젝트 메인 화면의 설계 계획서**입니다.

## 1. 아키텍처 핵심 목표
1. **컴포넌트 중심 설계 (Component-Driven)**: 재사용 가능한 Vue 파일 컴포넌트로 화면을 조립 (재사용성과 유지보수성 극대화).
2. **반응형 웹 디자인 (Responsive Web Design)**: SCSS의 중첩과 미디어 쿼리를 사용해, 자바스크립트 크기 체크 없이 순수 CSS(Flex/Grid)만으로 데스크톱(PC)과 모바일 뷰(가로 768px 이하) 간 끊김없는 레이아웃 전환.
3. **통합 검색의 전면 노출 (UX 개선)**: 사용자가 진입과 동시에 '목적지, 날짜, 인원'을 직관적으로 입력할 수 있는 대형 폼(Hero Search Bar) 배치.
4. **불필요한 팝업 제거 (Muted Interruptions)**: 화면을 가리는 불필요한 배너, 팝업 지양. 배너는 하단 혹은 스크롤 시 자연스럽게 전환되도록 배치.

---

## 2. 화면 단위(페이지) 구성도

전체 화면은 `pages/index.vue`에서 관리하며, 내부적으로 여러 단위 컴포넌트(`components/`)를 호출하여 조립합니다.

### 🧩 2.1 주요 컴포넌트 트리
*   **`Layout: Default.vue` (이미 구현됨)**
    *   **Header (GNB)**: 로고, 회원가입/로그인 (상태 연동 완료), 다국어 및 지역 메뉴.
    *   **Main 영역 (`<slot/>`)**: `pages/index.vue`
        *   ➡️ **`HeroSection.vue`**: 매력적인 배경 이미지 + 중앙 대형 **통합 검색창**
        *   ➡️ **`CategoryFilter.vue` (상단)**: 테마/카테고리 (가성비, 오션뷰, 한달살기, 펫동반 등) 아이콘 버튼 리스트
        *   ➡️ **`RecommendSection.vue` (중단)**: Pinia 인증 정보를 활용한 맞춤 추천 숙소 영역 (또는 트렌디 숙소)
        *   ➡️ **`RoomCard.vue` (반복자)**: 개별 숙소를 나타내는 카드 컴포넌트
    *   **Footer (`app-footer`)**: 사업자 정보 및 서비스 메뉴.

## 3. 핵심 컴포넌트 상세 설계 및 반응형(SCSS) 전략

### 🏙️ 3.1 `HeroSection.vue` (통합 검색 영역)
*   **기능**: 메인 배경 이미지 플레이 + [지역 검색창, 체크인/체크아웃, 인원 수 선택]을 하나로 합친 검색 Bar.
*   **반응형(SCSS) 전략**:
    - **PC (`width > 768px`)**: 검색 폼 요소들이 가로(`flex-direction: row`) 일직선 배열.
    - **모바일/웹뷰 (`width <= 768px`)**: 가로 공간 부족 시 `flex-direction: column`으로 세로 박스로 변환하고 클릭 영역 극대화.

### 🏘️ 3.2 `RoomCard.vue` (단일 숙소 미리보기 카드 컴포넌트)
여러 추천 리스트 영역에서 가장 많이 반복 호출되는 **공통 컴포넌트**입니다.

*   **반응형(SCSS) 전략 (CSS Grid 활용)**:
    - **PC**: `grid-template-columns: repeat(4, 1fr)` -> 4등분(`25%`) 차지. 화면 크기에 따라 썸네일 폭이 변동.
    - **태블릿 (`width <= 1024px`)**: `grid-template-columns: repeat(2, 1fr)` -> 가로 2개 배치 (`50%`)
    - **모바일/웹뷰 (`width <= 768px`)**: `grid-template-columns: repeat(1, 1fr)` -> 가로 1개 배치 (`100%`) + 썸네일 비율을 16:9 컨테이너에 맞게 강제. 상하 스크롤 유도.

## 4. 진행 계획 요약
1. `components` 디렉토리 하위에 공통 모듈 (예: `RoomCard.vue`, `HeroSection.vue`)을 생성합니다.
2. `pages/index.vue`를 열고, 위에 만든 컴포넌트를 `import`하여 메인 화면에 적절하게 배치합니다.
3. 디자인(스타일링) 작업이 완료되고 나면 백엔드(상품/방 API)와 연동하여 실제 데이터를 방 카드들에 뿌려 줍니다.
