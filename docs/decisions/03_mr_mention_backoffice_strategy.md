# 미스터멘션 백오피스(Back-Office) API 개발 제안 전략

## 배경 (Context)
미스터멘션은 제주도 등 **국내 한 달 살기(장기 숙박)** 플랫폼을 주력으로 하는 서비스입니다. 데일리 숙박 위주의 플랫폼(야놀자, 여기어때 등)과는 완전히 다른 라이프사이클을 가집니다. 일반적인 CRUD 게시판 수준이 아닌, 장기 체류에서 파생되는 다양한 리스크와 정산 문제, C/S(고객 서비스) 분쟁을 백오피스에서 효율적으로 통제할 수 있는 시스템 설계가 필수적입니다.

---

## 🚀 제안 API 아키텍처 및 핵심 기능 5선

### 1. 장기 숙박 특화: 보증금 및 분할 정산 API (Settlement API)
- **일반 플랫폼**: 체크아웃 시 1회 정산 후 종료.
- **미스터멘션 특수성**: 짧게는 1주, 길게는 수개월 장기 체류. 기간이 길기 때문에 공과금(수도/전기/가스) 차감 이슈와 보증금 관리가 매우 중요하며, 호스트의 자금 융통을 위한 중간 정산이 필요함.
- **제안 설계**:
  - **분할 정산 스케줄러**: 1주 또는 1개월 단위 등 호스트 설정에 따라 숙박비를 분할 지급할 수 있는 커스텀 정산 스케줄링 API.
  - **공과금 인증 및 예치금 차감 워크플로우**: 퇴실 시 호스트가 실비용 영수증(사진/PDF)을 백오피스에 업로드하면, CS 관리자가 이를 승인하여 게스트가 낸 보증금에서 차감한 후 남은 금액을 자동 환불해 주는 결제/정산 연동 API.

### 2. 장기 체류 복합 분쟁 조정 시스템 (CS / Resolution API)
- **특수성**: 일주일 이상의 체류는 호스트와 게스트 간 시설물 파손, 소음 분쟁, 예기치 않은 조기 퇴실 등의 마찰 확률을 높입니다.
- **제안 설계**:
  - **타임라인 기반 CS 티켓 생명주기 API**: 예약 1건에 종속된 분쟁 티켓을 생성해, `게스트 불만 접수 ➡ 증빙 자료 업로드 ➡ 관리자 중재 개입 ➡ 부분 환불 또는 대체 숙소 이동 승인`에 이르는 복잡한 상태(State) 전이를 관리하는 **State Machine** 패턴 기반 API 도입.

### 3. 사내 보안 체계 및 감사 로그 (RBAC & Audit Log API)
- **특수성**: 국내 장기 거주를 취급하므로 게스트의 개인정보(주민등록정보 일부, 긴급 연락처)와 호스트의 민감 정보(사업자 등록증, 정산용 은행 계좌명세)를 깊이 있게 다룹니다.
- **제안 설계**:
  - **세부적인 RBAC (역할 기반 접근 제어)**: 관리자 토큰 인가를 통제하여, CS팀은 환불과 예약 변경만 접근하고 영업팀은 숙소 승인 정보만 볼 수 있도록 세밀하게 권한을 분리한 API 게이트웨이(또는 Interceptor) 설계.
  - **AOP 기반 Audit Log (감사 로그)**: 어드민 내 모든 변경(Create/Update/Delete) API 호출 시 핵심 로직에 개입하지 않고 AOP를 활용하여 *"누가, 언제, 어떤 데이터의 무엇을 수정했는지(변경 전/후 페이로드)"*를 분리된 로그 DB에 강제 기록하여 내부 보안 통제.

### 4. 규제 샌드박스 컴플라이언스 및 제휴 플랫폼 연동 API (Compliance & Integration API)  *(🔥 강력 추천)*
- **특수성**: 미스터멘션은 내국인 공유숙박 실증 특례(연 180일 한도)를 독점적으로 활용해 에어비앤비와 '공동 호스팅' 파트너십을 체결했습니다. 이를 시스템적으로 감시하고 통제하는 기술적 대행 역할이 필수적입니다.
- **제안 설계**:
  - **영업일수 차감 파이프라인 및 대시보드 API**: 에어비앤비 등 타 플랫폼에서 넘어온 예약을 웹훅(Webhook)으로 수집해, 백오피스 내 관리자가 호스트의 '연간 180일' 한도를 실시간으로 모니터링하고 차감할 수 있는 컴플라이언스 대시보드 통합.
  - **내국인 예약 필터링 엔진**: 국가코드(+82) 및 한글 이름 판별을 통해 규제 특례 조건에 위배되는 예약을 식별하여 관리자에게 Alert를 발생시키는 기능.

### 5. 숙소 및 호스트 검수 워크플로우 (Moderation API)
- **특수성**: 장기 거주는 단기보다 비용 단위가 크기 때문에 허위/사기 매물이 등록되면 플랫폼 신뢰도에 치명타를 입습니다.
- **제안 설계**:
  - **파이프라인식 승인 API**: `호스트 가입 ➡ 필수 서류 제출(사업자/임대차계약서 등) ➡ 관리자 검토 ➡ 반려(사유 알림) ➡ 최종 승인`의 단계를 처리.
  - **자동 등급 관리(Batch) API**: 응답률, 예약 취소율, 리뷰 평점 등을 주기적으로 집계하여 호스트의 등급을 자동으로 강등시키거나 프로모션 대상으로 편입하는 백그라운드 스케줄링 통합.

### 6. 연박 할인 특화 다이나믹 마케팅 API (Promotion API)
- **특수성**: 숙박 일수(7박, 15박, 30박)에 따라 계단식으로 할인율이 달라지는 정책이 핵심입니다.
- **제안 설계**:
  - **Rule-based 쿠폰 발급 체계**: 관리자가 직접 타겟 계층을 식별하는 복합 룰(`"제주 지역 && 15박 이상 && 재구매 게스트 = 10% 추가 할인"`)을 백오피스 UI에서 만들면, 이를 서버 파싱 엔진에서 해석해 예약 시 동적으로 할인을 적용할 수 있는 마케팅 운영 엔진 API.

## � [Next Step] 백오피스 고도화 비전: ML/DL 기반 데이터 활용

단순한 데이터 조회를 넘어 플랫폼에 축적된 **숙박 통계와 유저 행동 데이터를 활용한 지능형 어드민(Intelligent Admin)**으로의 확장 계획을 제안합니다.

### 1. 장기 숙박 이탈 방지 예측 (Churn Prediction)
- **개념**: 사용자의 검색 및 찜하기 데이터를 머신러닝(RF, XGBoost)으로 분석하여 예약 이탈 가능성이 높은 사용자를 실시간 분류하는 분석 파이프라인.
- **백오피스 응용**: 어드민에서 '이탈 고위험군' 세그먼트를 추출하고 이들을 타겟으로 "시크릿 쿠폰(예: 15% 기습 할인)"을 자동/수동 발송하여 재결제 전환율을 방어합니다.

### 2. 분쟁(CS) 티켓의 NLP 기반 긴급도 스코어링
- **개념**: 한 달 살기 도중 접수되는 수많은 컴플레인(보일러 고장, 소음 등) 텍스트를 자연어 처리(NLP) 딥러닝 모델로 분석해 의도와 감정을 스코어링하는 기능입니다.
- **백오피스 응용**: 백오피스 CS 대시보드에 문의가 들어오자마자 모델이 **긴급도(Red/Yellow/Green)** 라벨을 자동으로 달아주어, 관리자가 중대형 분쟁을 초기에 진압할 수 있도록 운영 효율성을 극대화합니다.
