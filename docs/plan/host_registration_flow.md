# 📝 백오피스 특례 서류 업로드 페이지 구조 & 기획안

미스터멘션 공유숙박 실증특례 가이드북(PDF)을 분석하여 도출한 **B2B 백오피스(어드민) 호스트 특례 신청 페이지의 프론트엔드 UI/UX 흐름 및 상태 기획안**입니다.

## 1. 📂 페이지 단계 (Step-by-Step Flow)

사용자(호스트)가 회원가입 직후 B2B 백오피스(`admin.nextstay.com`)에 진입하면 가장 먼저 마주하게 되는 **'특례 신청(Onboarding)'** 온보딩 프로세스입니다.

### [Step 1] 사업자 유형(A안 / B안) 선택 화면
두 사업자의 요구 서류가 완전히 다르기 때문에 라디오 버튼으로 분기 처리를 합니다.
* **B안 (기존 외도민 사업자)**: "기존 외국인관광 도시민박업 사업자이신가요?" 
* **A안 (신규 비외도민 사업자)**: "신규로 공유숙박 특례를 시작하시는 분인가요?" (선택 시 180일 규제 및 2룸 이상 요건 팝업 안내)

### [Step 2-B] B안 분기 시서류 업로드 UI
기존에 지자체(구청) 허가를 받은 분들이므로 절차가 간단합니다.
1. `[S3 Upload Component]` **사업자등록증(세무서)** 파일 업로드 영역
2. `[S3 Upload Component]` **관광사업자등록증(구청)** 파일 업로드 영역
3. `[Text Input]` **에어비앤비 숙소 링크** 입력칸 (특례 문구 기재 확인용)
4. 약관(180일 관리, 보험 가입 등) 필수 체크박스

### [Step 2-A] A안 분기 시 서류 업로드 UI (심화)
새로 집을 오픈하는 분들이므로 강력한 서류 검증 UI가 필요합니다.
1. `[S3 Upload Component]` **건축물대장** (1층 상가 불가, 주택 용도 확인용)
2. `[S3 Upload Component]` **주민등록등본** (실거주 확인, 다가구 세대분리 확인용)
3. `[S3 Upload Component]` **등기부등본** (실소유주 파악용)
4. `[S3 Upload Component]` **건축물현황도(평면도)** (방 2개 + 거실 1개 최소 구조 충족 여부 육안 확인용)
5. `[S3 Upload Component]` **임대차계약서 & 임대인 동의서** (세입자일 경우 조건부 화면 표시)

### [Step 3] 최종 제출 및 심사 상태 조회(Dashboard)
제출(Submit) 버튼을 누르면 상태가 전환되며, 진행률 바(Progress Bar) 컴포넌트로 현재 과정을 보여줍니다.
1. `서류 대기/심사 중` -> 2. `현장 심사 예정` (소방안전시설 등 확인) -> 3. `안전 교육 이수` -> 4. `특례증 발급 완료` (최종 영업 개시)

## 2. ⚙️ 프론트엔드 - 백엔드 연동 아키텍처

대용량 이미지와 PDF 파일을 서버로 직행시키지 않고 트래픽을 최적화하는 **S3 Pre-signed URL** 패턴이 적용됩니다.

1. **사용자가 문서를 Drag & Drop** (예: 평면도.pdf)
2. **Vue.js 프론트엔드** ➡️ `POST /api/v1/files/presigned-url` 호출 (파일명 전달)
3. **Spring Boot 백엔드** ➡️ AWS S3로부터 일회용 업로드 티켓(URL) 획득 후 프론트엔드에 반환
4. **Vue.js 프론트엔드** ➡️ 발급받은 URL로 브라우저에서 **AWS S3로 직접 파일 원본을 전송 (비동기)**
5. **Vue.js 프론트엔드** ➡️ 모든 파일 업로드 완료 시, S3에 저장된 무거운 파일이 아닌 **파일의 주소(URL) 텍스트 리스트**만 모아 백엔드의 최종 특례 신청 API(`POST /admin/v1/applications`)로 전송
6. **DB (`applications` 테이블)** ➡️ 서류들의 S3 URL 경로와 신청 상태(Status = 'PENDING')를 저장.

이 구조를 통해 모바일에 익숙하지 않은 호스트들이 초고화질 휴대폰 카메라로 찍은 수십 MB의 등기부등본 사진을 올리더라도 메인 서버가 전혀 부하를 받지 않는 쾌적한 업로드 환경이 구축됩니다.
