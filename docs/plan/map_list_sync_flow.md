# 🗺️ 지도-리스트 동기화 (Map-List Sync) 프론트엔드 아키텍처

B2C (Guest 웹) 탐색 페이지에서 **숙소 리스트의 카드에 마우스를 올렸을 때(Hover), 옆의 카카오 지도(Kakao Map)에서 해당 숙소의 핀이 활성화**되며 가격과 정보가 뜨는 UI/UX의 기술적 구현 원리입니다.

## 1. 아키텍처 원리: Data Attribute와 전역 상태(State) 공유

프론트엔드(Vue.js / React)의 가장 강력한 특징인 **'상태 기반 렌더링(State-driven Rendering)'**을 완벽하게 보여주는 예시입니다. 이 로직은 백엔드 API를 추가로 호출하지 않고, **프론트엔드 내부의 상태(메모리)만으로 즉시 동작(0.01초)**해야 합니다.

### [동작 흐름]
1. **API 최초 호출**: 유저가 특정 지역을 검색하면, 백엔드는 해당 지역의 숙소 리스트 JSON 배열을 던져줍니다. (이 JSON 안에는 숙소의 `lat`(위도), `lng`(경도), `price_month`(가격), `title`(숙소명) 등이 모두 포함되어 있습니다.)
2. **DOM에 데이터 은닉 (`data-set`)**: 
   프론트엔드는 이 리스트를 화면에 반복문(v-for / map)으로 그리면서, 각 숙소 카드의 가장 바깥쪽 `<div data-set='{...}'>`에 숙소의 원본 JSON 데이터를 숨겨둡니다.
3. **이벤트 리스너 (Hover / MouseEnter)**: 
   유저가 6907번 숙소 카드에 마우스를 올리면(`mouseenter` 이벤트 발생), 프론트엔드는 해당 div에 숨겨진 `data-set` 값(특히 `lat, lng, price`)을 읽어냅니다.
4. **상태(State) 업데이트**:
   읽어낸 데이터를 즉시 **뷰의 전역 상태관리자(Vuex / Pinia) 안의 `hoveredAccommodation` 변수에 저장**합니다.
   ```javascript
   // Pinia Store 예시
   state: () => ({
     hoveredAccommodation: { id: 6907, lat: 33.323244, lng: 126.171452, price: 2804520 }
   })
   ```
5. **지도 컴포넌트 반응(Reactivity)**:
   화면 좌측의 카카오맵 컴포넌트는 Pinia의 `hoveredAccommodation` 상태를 구독(Subscribe)하고 있습니다. 이 값이 방금 클릭한 6907번 숙소로 변하는 순간, 카카오맵 SDK의 로직이 발동하여 `lat, lng` 좌표로 **지도 중심을 스윽 부드럽게(PanTo) 이동**시키고, 해당 핀 위에 **커스텀 오버레이(Custom Overlay)**를 띄워 숙소 이름과 가격("2,804,520원")을 렌더링합니다!

## 2. 면접 어필을 위한 설계 철학

이 구조를 포트폴리오 면접에서 설명할 때 다음과 같이 어필하면 프론트엔드 역량을 극대화할 수 있습니다.

> **"마우스 오버나 스크롤 시마다 백엔드 API를 다시 찌르는 것은 서버에 엄청난 트래픽 폭탄(DDoS)을 유발하는 안티 패턴입니다. 따라서 저는 최초 검색 시 백엔드로부터 한 번 받아온 JSON 데이터를 프론트엔드의 `data` 속성으로 컴포넌트에 바인딩 해두고, 마우스 Hover 시 이 데이터만 추출하여 Pinia(전역 상태)에 올리는 방식을 채택했습니다. 
> 좌측의 카카오맵 컴포넌트는 이 Pinia 상태를 지켜보다가 값이 바뀌면 즉시 카카오맵 SDK의 `panTo()` 명령어를 호출해 지도를 이동시키므로, 서버 부하는 '제로(0)'이면서 유저에게는 완벽하게 부드럽고 쾌적한 UX를 제공하는 프론트엔드 최적화를 달성했습니다."**
